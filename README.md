# 微信聊天记录读取器 (WeChat Recorder)

一个基于 Windows UI 自动化技术的微信聊天记录提取工具。通过模拟用户操作（定位元素、模拟点击、剪贴板复制），自动从 PC 端微信窗口中提取聊天记录。

## ⚠️ 核心原理与限制

本项目**不使用** OCR（光学字符识别）或图像识别技术，而是通过 Windows Accessibility API 和剪贴板操作来获取数据。

### 安全声明
本工具仅用于**个人数据备份**。它通过模拟标准的用户交互（鼠标滚动、Ctrl+C）来运行，**不涉及**任何协议逆向、内存注入或二进制修改。从技术检测层面看，它的行为与正常人类用户手动浏览和复制没有本质区别。

但为了最大程度降低账号风险，请遵循以下原则：
1.  **不要全天候运行**：避免连续数小时的高频操作。
2.  **不要用于群发或营销**：本工具仅用于“读取”和“备份”。
3.  **保持“拟人化”设置**：代码中已内置随机延迟，请勿随意移除。

### 优势
- **准确性高**：直接获取文本内容，不存在 OCR 识别错误。
- **依赖轻量**：无需安装 Tesseract 或庞大的机器学习库。
- **速度快**：无需进行复杂的图像处理计算。

### 局限性
1.  **独占性强**：程序运行时会控制鼠标和键盘，且频繁使用剪贴板。**建议在运行时不要操作电脑**。
2.  **窗口必须可见**：微信窗口必须保持在前台，不能被遮挡或最小化。
3.  **UI 结构依赖**：如果微信大幅更新 UI 结构，可能需要调整元素定位逻辑。

## 核心架构

### 1. 自动化控制层 (Automation Layer)
- **uiautomation**: 核心库。用于遍历微信窗口的 UI 树，定位消息列表、输入框、联系人等元素。
- **pyautogui**: 辅助库。用于处理无法通过 UI 树直接交互的区域，进行模拟点击或滚动。
- **pywin32**: 窗口句柄管理，确保窗口置顶和激活。

### 2. 数据获取层 (Extraction Layer)
- **Clipboard Strategy**: 对于无法直接读取 `Value` 属性的控件，通过模拟 `Ctrl+C` 将内容复制到剪贴板，再通过 `pyperclip` 读取。
- **Element Parsing**: 直接解析 UI 元素的 `Name` 或 `LegacyIAccessible.Value` 属性获取文本。

### 3. 数据存储层 (Data Layer)
- **Pandas**: 将提取的聊天记录（发送人、时间、内容、类型）结构化为 DataFrame。
- **Storage**: 支持导出为 CSV 或 Excel。

## 安装与配置

### 环境要求
- Windows 10/11
- Python 3.9+
- PC 端微信（建议保持最新版）

### 安装依赖
推荐使用 `uv` 进行包管理：

```bash
# 安装 uv (如果尚未安装)
pip install uv

# 创建虚拟环境并同步依赖
uv venv
uv pip install -r requirements.txt
```

## 使用方法

1.  **启动微信**：登录 PC 微信，并打开要提取的聊天窗口。
2.  **运行程序**：
    ```bash
    python -m src.main
    ```
3.  **操作提示**：
    - 程序启动后会自动寻找微信窗口。
    - **请双手离开键盘鼠标**，程序将自动滚动并抓取记录。
    - 按 `Ctrl+C` 强制停止。
